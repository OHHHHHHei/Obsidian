# String Interpolation

字符串插值，简单来说，就是**将变量、常量或表达式的值嵌入到一个字符串模板中，从而动态地创建一个新字符串**。这让我们可以用一种非常自然和可读的方式来构建包含动态内容的字符串。

## F-string 

在字符串的起始引号前加上一个字母 $f$ 或 $F$。然后在字符串内部，用花括号 `{}` 把要插入的变量或表达式包起来。Python会自动解析`{}` 里面的值并且插入进去

# Implementing repr and srt

Python 的内置函数 (如 `repr()`, `str()`, `len()`) 在调用对应的特殊方法 (如 `__repr__`, `__str__`, `__len__`) 时，**会绕过实例属性，直接去对象的类定义中查找**。

正确的实现方式是使用 `type(x)` 来获取 `x` 的类，然后调用在类上定义的 `__repr__` 方法，并将 `x` 作为 `self` 参数传递进去，即 `type(x).__repr__(x)`。

# Interfaces

在 Python 中，"接口" 不是一个严格的语法，而是一种**非正式的协议 (protocol) 或契约**。

一个 "接口" 就是一套大家约定俗成的 "共享消息"（即方法名，如 `__repr__` 和 `__str__`）。任何类，只要它正确地实现了这套方法（并遵循了它们的含义规范），我们就说这个类 "实现了" (implements) 或 "遵循了" (adheres to) 这个接口。

这就是为什么 `str()` 和 `repr()` 这样的**多态函数**能工作的原因：它们不在乎你传入的是什么类型的对象，它们只在乎 "这个对象是否实现了'字符串表示接口'？" 如果实现了，`str()` 就去调用它的 `__str__`，`repr()` 就去调用它的 `__repr__`。

# Special Method Names in Python

- **什么是特殊方法？**
    - 它们是 Python 预定义好的一些方法，名字**总是以双下划线开始和结束**（例如 `__init__`, `__add__`, `__repr__`）。
    - 它们之所以"特殊"，是因为你**通常不会直接调用**它们。相反，是 Python 的**内置操作** (比如运算符、内置函数) 会在特定时机**自动调用** (invoke) 它们。
- **有什么用？ (运算符重载)**
    - 它们的主要用途是让**你自己定义的类 (Class)** 能够响应和模仿 Python 的内置行为。
    - 例如，通过在你的类中定义 `__add__` 方法，你就能控制当你的对象（实例）被用于 `+` 运算符时，应该发生什么。这个就叫做 **"运算符重载" (Operator Overloading)**。
